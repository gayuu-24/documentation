Error Handling

Error handling primarily involves detecting and managing exceptions to ensure a program can respond to unexpected situations gracefully. There are three main types of errors in Python

  • Syntax Errors
  • Logical Errors
  • Runtime Errors

Syntax Errors

Syntax Errors occur when the Python interpreter encounters invalid code structure, such as missing colons, mismatched parentheses, or incorrect indentation.

These errors prevent the program from running and must be fixed before execution.

Example:

if True

   print("Hello")

Output:

SyntaxError: expected ':'

Fix the code syntax before running the program. Syntax errors cannot be caught with a try-except block because the code doesn't execute at all.

Logical Errors

These occur when the program runs without crashing but produces incorrect results due to flaws in the program logic.

Example:

def add_numbers(a, b):

   return a - b         # Logical error, should be a + b

print(add_numbers(5, 3))

Output:

2 (incorrect)

Logical errors require careful debugging, reviewing the logic, and writing test cases to catch flaws.

Run-time Errors

These occur while the program is running and disrupt normal flow. They happen when the program encounters an operation it cannot execute.

Its like users mistake of giving wrong inputs.

Example:

a=int(input())

b=int(input())

print(a+b)

Output:

10

hi

ValueError: invalid literal for int() with base 10: 'hi'

To overcome this error we can use try-except technique in this program.

After adding try-except in program:

try:

      a=int(input())

      b=int(input())

      print(a+b)

except Exception:

      print("error in input")

Output:

10

hi

error in inpurt

Explanation:

This code attempts to take two integer inputs, a and b, and prints their sum.

If any exception occurs during input conversion or addition (e.g., entering non-integer values), the except block catches it and prints "error".

The Exception class is a base class for most errors in Python, making this a generic error handler.
